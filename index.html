Project Directory Structure/Hierarchy

  • Requirements
    • the conceptual description of the application
    • the description, goal, target users, and all other information you would give an investor or partner
  • Design
    • a middle ground between the Requirements and the Implementation
    • more abstract than the implementation, but less abstract than the Requirements
    • the blueprints of how to implement the program
    • the reasonably detailed specification of important subsystems and classes, the user interface, and how the program and the user interact
  • Implementation
    • the implementation includes the source code, user documentation, programmer documentation, and distributable program
  • Administration
    • the current state of the project, goals, todos, and other meta-documents
